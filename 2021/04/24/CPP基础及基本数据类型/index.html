C++基础及基本数据类型

Posted on 2021-04-24 Edited on 2021-05-18 In C++

开始学习C++

进入C++

预处理器编译指令 #include

编译指令 using namespace

main()被启动代码调用，而启动代码是编译器添加到程序中的。函数头描述的是main()和操作系统之间的接口

没有h的头文件也可以包含命名空间（？）

命名空间

  1. using namespace std;

  2. using std::cout

  3. std::cout

当前通行的理念是，只让需要访问名称空间std的函数访问它是更好的选择

对于cout << "C++ RULES"：

  1. cout对象

  2. <<插入运算符

  3. "C++ RULES"被插入的字符串

endl确保程序继续运行前刷行输出

C++语句

对于声明变量，C++的做法是尽可能在首次使用变量前声明它

其他C++语句

C++提供了两种发送消息的方式：一种是使用类方法，另一种是重新定义运算符。

函数

原型

C++程序应当为程序中使用的每个函数提供原型

原型结尾的分号表明它是一条语句，这使得它是一个原型，而不是一个函数头

用两种方法来实现：

  1. 在源代码文件中输入函数原型
  2. 包含头文件cmath，其中定义了原型

原型只描述函数接口，定义中包含了函数的代码。库文件包含了函数的编译代码，头文件包含了原型。通常把原型放到main()函数定义的前面

对于用户定义的函数，通常把原型放到main()函数定义的前面，把代码放到main()的后面

C++不允许将函数定义嵌套在另一个函数定义中

退出值为0说明程序运行成功，非0则意味着存在问题

处理数据

简单变量

变量名

以两个下划线开头或一个下划线和一个大写字母开头的变量名被保留给实现，使用会导致行为的不确定性。以一个下划线开头的名称用作全局标识符

整型

  1. short至少16位
  2. int至少与short一样长
  3. long至少32位，且至少与int一样长
  4. longlong至少64位，且至少和long一样长

使用sizeof运算符时，若是类型名，应将其放在括号中，但对变量名，括号是可以选择的。

C++字节由至少能够容纳实现的基本字符集（ASCII、Unicode等）的相邻位组成。

<climits>中定义了符号常量。

C++11的初始化方式

无符号类型

C++并不保证符号整型超越限制（上、下溢）

如果没有有说服力的理由，则应该使用int。如果变量表示的整数值可能大于16位，则应该使用long，否则移植到16位系统的时候可能导致错误。

当且仅当大型整型数组时，才有必要使用short

整型字面值

十进制：第一位为1-9；八进制：第一位为0，第二位为1-7；十六进制：0x或者0X开头

输出：cout提供了控制符dec， hex， oct，分别用于指示十进制、十六进制和八进制

常量类型

  1. 后缀：l/L, u/U, ul/UL/lu/LU, ll/LL, ull…
  2. 长度：
    对于十进制，依次用int、long、long long；对于八进制、十六进制，依次用int, unsigned int, long, ul, ll, ull。十六进制常用来表示地址，没有符号。

char类型

可以用字符的八进制或十六进制编码来使用转义序列

（这里没有看得很懂）通用字符名：源字符集、执行字符集；扩展

char是否有符号与实现有关，可以显示定义。如果将char视作数值类型，则unsigned 和signed的差异就比较重要。

wchar_t类型（宽字符类型），对底层类型的选择取决于实现。

C++11新增的类型：char16_t, char32_t.

const限定符

在声明中对const进行初始化。

C++算术运算符

当两个运算符被用于一个操作数时，考虑优先级和结合性。第三行究竟先算哪个由实现决定。

除法运算符的行为取决于操作数的类型：整/整=整；有一个为浮点数则结果为浮点数。

初始化时的类型转换

类型转换可能导致精度降低、小数部分丢失、只复制右边的字节等。

以{}方式进行初始化时的转换要求严格，不允许缩窄(narrowing)。

表达式中的转换

整型提升。bool,unsigned char,signed char,short转化为int;

如果short和int一样长，unsigned short将被转化成无符号的整型。

C++11的校验表：

  1. 浮点型转化成较长的类型
  2. 整型进行整型提升
  3. 都是有/无符号的，转化成级别高的类型
  4. 一个有符号一个无符号，无符号级别高，转化成无符号
  5. 否则，有符号类型可以表示无符号类型的所有可能取值，转换为有符号
  6. 转换为有符号类型的无符号版本
